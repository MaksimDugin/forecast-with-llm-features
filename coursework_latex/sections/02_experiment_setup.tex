\section{Экспериментальная часть}

\subsection{Постановка задачи}

Эмпирическая часть работы направлена на проверку того, как разные группы признаков влияют на прогноз направления движения цены финансового актива. В отличие от задачи точечного прогнозирования цены, далее рассматривается задача бинарной классификации направления: необходимо определить, окажется ли будущая цена закрытия выше текущей.

Пусть $a$ --- финансовый актив, а $S_t^{(a)}$ --- цена закрытия этого актива в момент $t$. Для фондового рынка индекс $t$ соответствует торговым дням, а для криптоактивов --- дневным календарным наблюдениям. Для горизонта прогноза $h$ целевая переменная задаётся как

\[
Y_{t,h}^{(a)} = \mathbf{1}\{S_{t+h}^{(a)} > S_t^{(a)}\}.
\]

Если цена закрытия через $h$ дней выше текущей цены закрытия, наблюдению присваивается класс $1$; если цена не выросла, присваивается класс $0$. В основном эксперименте для AAPL используется горизонт $h=1$, то есть прогноз направления на следующий торговый день. В дополнительной мультиактивной проверке для криптоактивов рассматриваются горизонты $h \in \{1,3,7\}$.

Для каждого наблюдения строится вектор признаков

\[
X_{t,h}^{(a)} = \bigl[X_{market,t}^{(a)}, X_{text,t}^{(a)}\bigr],
\]

где $X_{market,t}^{(a)}$ обозначает рыночные признаки, построенные по ценам, доходностям и объёмам торгов, а $X_{text,t}^{(a)}$ --- текстовые признаки, извлечённые из финансовых текстов с помощью FinBERT или производных текстовых представлений. Отдельные эксперименты используют разные подмножества этого вектора: только рыночные признаки, только текстовые признаки или их объединение.

Задача модели состоит в построении отображения

\[
f_\theta: X_{t,h}^{(a)} \mapsto \widehat{p}_{t,h}^{(a)},
\qquad
\widehat{p}_{t,h}^{(a)} = \mathbb{P}\left(Y_{t,h}^{(a)}=1 \mid X_{t,h}^{(a)}\right),
\]

где $\widehat{p}_{t,h}^{(a)}$ интерпретируется как оценка вероятности положительного класса. Бинарный прогноз получается после выбора порога классификации $\tau$:

\[
\widehat{Y}_{t,h}^{(a)} = \mathbf{1}\{\widehat{p}_{t,h}^{(a)} \geq \tau\}.
\]

Для финансовых временных рядов принципиально важно, чтобы признаки были доступны до момента прогноза. Это условие можно записать через фильтрацию $(\mathcal{F}_t)_{t \geq 0}$, где $\mathcal{F}_t$ обозначает информацию, доступную к моменту $t$:

\[
X_{t,h}^{(a)} \in \mathcal{F}_t,
\qquad
Y_{t,h}^{(a)} \in \mathcal{F}_{t+h}.
\]

Следовательно, при построении $X_{t,h}^{(a)}$ нельзя использовать цены, тексты или статистики, появившиеся после момента принятия решения. В работе это требование реализуется через хронологическое разбиение данных и через обучение всех преобразований признаков только на обучающей подвыборке.

Главный исследовательский вопрос экспериментальной части состоит в следующем: дают ли признаки, извлечённые из финансовых текстов с помощью FinBERT и связанных текстовых представлений, самостоятельный сигнал и добавочную предсказательную силу относительно рыночных признаков. Формально это проверяется через сравнение качества моделей на разных группах признаков при едином протоколе обучения и оценки.

\subsection{Данные}

Эксперименты используют несколько источников данных. Основной эксперимент проводится на акции AAPL и служит базовой проверкой влияния FinBERT embeddings. Дополнительный эксперимент с расширенным рыночным baseline также проводится на AAPL, но использует другой набор данных и более детальное разбиение признаков. Мультиактивная проверка расширяет постановку на криптоактивы и новостной датасет \texttt{crypto-news}.

\subsubsection{Данные основного эксперимента AAPL}

В основном эксперименте выбран актив AAPL. Данные имеют дневную частоту, а горизонт прогноза составляет один торговый день. Итоговый набор данных основного эксперимента содержит 1578 row-level наблюдений и 117 уникальных дат. В RUN 03 строка датасета соответствует не только дате принятия решения, но и текстовому объекту, связанному с этой датой. Поэтому несколько строк могут относиться к одной и той же дате, а метрики основного эксперимента являются row-weighted.

Целевая переменная задаётся как

\[
Y_{t,1}^{AAPL}=\mathbf{1}\{S_{t+1}^{AAPL}>S_t^{AAPL}\}.
\]

Для контроля утечки данных используется хронологическое разбиение по дате принятия решения. Одна и та же дата не может попадать одновременно в обучающую, валидационную и тестовую подвыборки. Основной эксперимент RUN 03 использует разбиение \texttt{chronological\_row\_balanced\_by\_decision\_date}.

Такое разбиение сохраняет временную структуру данных, но не устраняет различие между row-level и date-level оценкой. На тестовой подвыборке 222 наблюдения соответствуют 55 уникальным датам, поэтому даты с большим числом текстов получают больший вес в row-weighted метриках. Для финансовой интерпретации результата требуется дополнительная date-weighted проверка или агрегация текстовых признаков до одного объекта на дату принятия решения. Текстовые признаки используются с лагом: для прогноза после дня $t$ учитываются тексты не позднее $t-1$. Это правило снижает риск попадания в модель информации, которая фактически появилась после прогнозируемого события.

\subsubsection{Данные дополнительного эксперимента с расширенным рыночным baseline}

Дополнительный эксперимент RUN 04 также связан с AAPL, но использует данные \texttt{TheFinAI/flare-sm-bigdata} и более детальную схему признаков. В нём отдельно сравниваются исходные ценовые признаки, rolling-признаки, признаки торгового объёма, полный FinBERT embedding и PCA-сжатые текстовые представления. Итоговый датасет RUN 04 содержит 163 наблюдения и 163 уникальные даты за период с 2020-03-09 по 2020-10-27. Более подробное описание этого эксперимента приведено в разделе 2.7.

Смысл RUN 04 состоит в том, чтобы проверить, не является ли эффект текстовых признаков в основном эксперименте следствием слабости исходной рыночной базовой модели. Поэтому в нём усиливается рыночный baseline и отдельно проверяется, сохраняется ли добавочный вклад FinBERT embeddings после снижения размерности. С учётом малого размера тестовой подвыборки RUN 04 интерпретируется как exploratory robustness check.

\subsubsection{Данные мультиактивной проверки crypto-news}

Мультиактивная проверка RUN 05 использует новости из датасета \texttt{crypto-news} и дневные OHLCV-данные Binance. В отличие от AAPL-экспериментов, здесь рассматриваются криптоактивы BNB, BTC и SOL. Для каждого актива строятся задачи прогноза на горизонтах $h \in \{1,3,7\}$ календарных дней:

\[
Y_{t,h}^{crypto}=\mathbf{1}\{S_{t+h}^{crypto}>S_t^{crypto}\}.
\]

Криптовалютный рынок торгуется непрерывно, поэтому здесь используется не понятие следующего торгового дня, а календарный дневной горизонт. Тексты агрегируются и выравниваются с дневными ценами так, чтобы признаки соответствовали информации, доступной к моменту прогноза.

Поскольку в этом эксперименте сравнивается несколько активов, горизонтов и групп признаков, его результаты следует интерпретировать как exploratory robustness check: они показывают возможные области, где текстовые признаки дают заметный прирост, но требуют дальнейшей проверки на независимых периодах или в rolling-window схеме.
