\subsection{Методология сравнения групп признаков}

Экспериментальная методология строится как сравнение групп признаков при едином протоколе обучения и оценки. Это означает, что различия в результатах должны интерпретироваться прежде всего как различия в информативности входных признаков, а не как следствие разных условий эксперимента. Такой подход соответствует ablation-логике: вклад новой группы признаков оценивается через изменение качества модели при её добавлении или удалении \cite{guyon2003feature}.

Основные группы признаков сравниваются в задаче бинарной классификации направления движения цены. Для актива $a$ и горизонта $h$ используется целевая переменная

\[
Y_{t,h}^{(a)} = \mathbf{1}\{S_{t+h}^{(a)} > S_t^{(a)}\}.
\]

В основном эксперименте для AAPL рассматривается $h=1$. В мультиактивной проверке RUN 05 рассматриваются горизонты $h \in \{1,3,7\}$ для криптоактивов. Такая запись позволяет использовать одну формальную постановку для всех экспериментов.

Разбиение данных выполняется хронологически. Сначала модель обучается на обучающей подвыборке, затем гиперпараметры и порог классификации выбираются на валидационной подвыборке, а итоговая оценка проводится на тестовой подвыборке. Одна и та же дата не может попасть одновременно в разные подвыборки. Это важно для финансовых временных рядов, поскольку случайное перемешивание наблюдений может привести к утечке информации между близкими датами. В обзоре литературы такая проблема отмечается как одно из слабых мест части работ по текстовому прогнозированию финансовых рядов \cite{hiew2019bert,jiang2023finbert}.

Все гиперпараметры выбираются только на валидационной подвыборке. Тестовая подвыборка используется только для финальной оценки уже выбранной конфигурации. Это правило относится к параметрам логистической регрессии, порогу классификации, размерности PCA-представления и выбору групп признаков в confirmatory-постановке. Если какая-либо конфигурация выбирается по тестовому результату, такой запуск должен интерпретироваться как exploratory, а не как окончательная статистическая проверка.

Нормализация рыночных признаков выполняется с помощью \texttt{StandardScaler}. Scaler обучается только на обучающей подвыборке, после чего то же преобразование применяется к валидационной и тестовой подвыборкам. Для признака $z$ это можно записать как

\[
\tilde z = \frac{z - \mu_{train}}{\sigma_{train}},
\]

где $\mu_{train}$ и $\sigma_{train}$ рассчитываются только по обучающей выборке. Такой порядок нужен для предотвращения утечки данных из будущих периодов в предварительную обработку. То же правило применяется к PCA и другим преобразованиям признаков: операция \texttt{fit} выполняется только на обучающей подвыборке, а к валидационной и тестовой подвыборкам применяется только \texttt{transform}.

Метрики качества рассчитываются на тестовой подвыборке. Для оценки используются accuracy, balanced accuracy, precision, recall, F1-score, ROC-AUC и confusion matrix. Accuracy показывает общую долю правильных предсказаний:

\[
Accuracy = \frac{TP + TN}{TP + TN + FP + FN}.
\]

При несбалансированном распределении классов accuracy может быть недостаточной, поэтому дополнительно используется balanced accuracy:

\[
Balanced\ Accuracy = \frac{1}{2}\left(\frac{TP}{TP + FN} + \frac{TN}{TN + FP}\right).
\]

F1-score объединяет precision и recall:

\[
F1 = 2 \cdot \frac{Precision \cdot Recall}{Precision + Recall}.
\]

ROC-AUC используется как метрика ранжирования, показывающая, насколько модель отделяет положительный класс от отрицательного при разных порогах классификации. Поэтому наиболее убедительными считаются случаи, где улучшение наблюдается не только по threshold-dependent метрикам, но и по ROC-AUC.

Добавочная предсказательная сила текстовых признаков измеряется через изменение метрики относительно рыночной модели:

\[
\Delta_{text}^{M}
=
M(X_{market},X_{text}) - M(X_{market}),
\]

где $M$ --- выбранная метрика качества, $X_{market}$ --- рыночные признаки, а $X_{text}$ --- одна из текстовых групп признаков. Если $\Delta_{text}^{M} > 0$, добавление текстовых признаков улучшает результат относительно рыночной группы по метрике $M$. Если $\Delta_{text}^{M} \leq 0$, добавочная предсказательная сила текстовых признаков в данном протоколе не подтверждается.

Методология сравнения строится вокруг следующих вопросов:

\begin{enumerate}
    \item содержат ли текстовые признаки самостоятельный сигнал относительно \texttt{majority baseline};
    \item дают ли рыночные признаки прогнозный сигнал относительно \texttt{majority baseline};
    \item улучшают ли FinBERT embeddings качество прогноза при добавлении к базовым рыночным признакам;
    \item сохраняется ли добавочная сила текстовых признаков относительно более сильного \texttt{market\_full} baseline;
    \item зависит ли эффект текстовых признаков от актива и горизонта прогноза;
    \item помогает ли более сложная последовательная модель извлекать дополнительный сигнал из объединённых признаков.
\end{enumerate}

Отдельно следует учитывать проблему множественного сравнения. При одновременном сравнении нескольких активов, горизонтов и групп признаков максимальный найденный прирост может быть смещён вверх. Поэтому RUN 05 интерпретируется как поиск кандидатов на наличие текстового сигнала, а не как окончательное статистическое доказательство. Найденный максимум по BNB требует дальнейшей проверки на независимом периоде, через rolling-window validation или с использованием статистических процедур контроля множественных сравнений.

Таким образом, дизайн экспериментов соответствует feature-centric логике исследования: модели используются как инструмент проверки признаков, а не как самостоятельный объект оптимизации. Главный результат оценивается не по абсолютному максимуму метрики, а по тому, даёт ли новая группа признаков устойчивый прирост относительно корректного рыночного baseline.
