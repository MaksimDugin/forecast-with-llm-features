\subsection{Интерпретация и ограничения}

Полученные результаты необходимо интерпретировать в логике исследования признаков, а не в логике соревнования моделей. Цель экспериментов состояла не в подборе архитектуры с максимальной точностью, а в проверке того, содержат ли LLM/FinBERT-generated features самостоятельный сигнал и дают ли они добавочную предсказательную силу относительно рыночных признаков. Такой подход согласуется с литературой по feature selection и feature contribution: новая группа признаков имеет смысл только тогда, когда её добавление улучшает качество прогноза при сопоставимом протоколе оценки \cite{guyon2003feature}.

Формально добавочную силу текстовых признаков можно определить через изменение выбранной метрики качества:

\[
\Delta_{text}^{M} = M(X_{market}, X_{text}) - M(X_{market}),
\]

где $M$ --- метрика качества, $X_{market}$ --- рыночные признаки, а $X_{text}$ --- текстовые признаки, полученные с помощью FinBERT или других LLM-derived представлений. При такой записи положительное значение $\Delta_{text}^{M}$ означает, что добавление текстовых признаков улучшает результат относительно модели на рыночных признаках. Однако для устойчивого вывода желательно, чтобы улучшение наблюдалось не только по одной threshold-dependent метрике, но и по ROC-AUC или на нескольких активах и горизонтах.

Результаты основного эксперимента RUN 03 дают слабоположительный, но неустойчивый сигнал. Для accuracy и balanced accuracy добавление FinBERT-признаков к рыночным признакам улучшает слабую модель \texttt{prices-only}:

\[
\Delta_{text}^{Accuracy} = 0.486 - 0.410 = 0.076,
\]

\[
\Delta_{text}^{Balanced\ Accuracy} = 0.474 - 0.400 = 0.074.
\]

Этот результат указывает на наличие слабого, но неустойчивого текстового сигнала: при объединении с ценовыми признаками несколько метрик растут относительно простого price baseline. Такой результат согласуется с общей идеей textual finance, согласно которой тексты могут содержать информацию о настроениях, ожиданиях и интерпретации событий \cite{tetlock2007media,loughran2011liability,kearney2014textual,nassirtoussi2014text}.

Однако основной эксперимент RUN 03 не даёт сильного подтверждения добавочной силы. Объединённая модель \texttt{prices + FinBERT} не превосходит \texttt{majority baseline} по accuracy, уступает \texttt{FinBERT-only}, а ROC-AUC объединённой модели остаётся ниже 0.5:

\[
ROC\text{-}AUC_{combined} = 0.427 < 0.5.
\]

Следовательно, в этом протоколе не подтверждается устойчивая комплементарность рыночных и текстовых признаков. Текстовые embeddings демонстрируют слабый самостоятельный сигнал, но их объединение с рыночными признаками не приводит к надёжному улучшению качества.

Дополнительный эксперимент 1 RUN 04 усиливает осторожность этого вывода. После расширения рыночного baseline лучшей группой становится \texttt{market\_full}: balanced accuracy равна 0.577, ROC-AUC равен 0.853. Добавление PCA-сжатых FinBERT embeddings к \texttt{market\_full} не улучшает accuracy и balanced accuracy, а ROC-AUC немного снижается. Это показывает, что часть видимого эффекта текстовых признаков в основном эксперименте могла быть связана со слабостью исходного price baseline. Когда baseline включает rolling-динамику и признаки объёма, устойчивое преимущество FinBERT embeddings для AAPL не подтверждается.

Дополнительный эксперимент 2 RUN 05 меняет интерпретацию с отрицательной на условную. В мультиактивной crypto-постановке текстовые признаки дают заметный прирост для отдельных активов и горизонтов. Наиболее сильный результат получен для BNB: на горизонте 1 день добавление FinBERT-derived и text-meta признаков к \texttt{market\_full} повышает balanced accuracy с 0.498 до 0.615 и ROC-AUC с 0.476 до 0.670. На горизонте 7 дней для BNB balanced accuracy увеличивается с 0.415 до 0.569. Для SOL прирост balanced accuracy также положителен на всех трёх горизонтах, но менее устойчив по ROC-AUC. Для BTC эффект слабее и почти исчезает на горизонте 7 дней.

Для самого сильного случая, BNB на горизонте 1 день, была дополнительно проведена локальная статистическая проверка устойчивости прироста. Сначала использовался paired bootstrap по датам тестовой подвыборки. В каждой bootstrap-итерации $b$ пересэмплировались даты, после чего вычислялась разность метрик:

\[
\Delta_{b}^{M}=M_b(X_{market},X_{text})-M_b(X_{market}).
\]

Полученные интервалы в основном лежат выше нуля. Для balanced accuracy нижняя граница 95\% bootstrap-интервала равна 0.005, а для ROC-AUC --- 0.041. Это усиливает интерпретацию BNB 1d как кандидата на наличие текстового сигнала. При этом результат остаётся локальным: он относится к одному активу, одному горизонту и одной выбранной конфигурации признаков.

В качестве дополнительного sanity check была проведена проверка с перестановкой дат текстовых предсказаний. В этой проверке сохранялись истинные значения целевой переменной и рыночная модель, но scores модели с текстовыми признаками перемешивались между датами:

\[
X^{perm}_{text,t}=X_{text,\pi(t)}.
\]

Идея проверки состоит в том, что если эффект связан только с маргинальным распределением scores, а не с временным выравниванием текстового сигнала, то после перестановки прирост должен сохраняться. Результаты 2000 перестановок приведены в таблице~\ref{tab:run05_bnb_permutation_check}.

После перестановки фактический прирост оказывается выше почти всех случайно перемешанных вариантов: только 0.45\% перестановок дают прирост balanced accuracy не ниже фактического, а для ROC-AUC аналогичная доля составляет 0.60\%. Это является дополнительным аргументом в пользу того, что результат BNB 1d связан с временно согласованным текстовым сигналом. Однако данная проверка является prediction-level sanity check, а не полной permutation-процедурой с переобучением модели на перемешанных текстовых признаках.

Таким образом, итоговый вывод состоит не в том, что текстовые признаки бесполезны, и не в том, что они универсально улучшают прогноз. Результаты показывают неоднородность эффекта: вклад LLM/FinBERT-generated features зависит от актива, источника текстов, горизонта прогноза, способа построения текстовых признаков и силы рыночного baseline. Такой вывод согласуется с прикладными работами по финансовому NLP, где результат часто зависит от источника сообщений, целевой переменной и рынка \cite{hiew2019bert,zou2022prebit,jiang2023finbert,albada2025bertweet,vuong2025llm}.

Отдельно следует интерпретировать различие между ROC-AUC и threshold-dependent метриками. Accuracy, F1 и balanced accuracy зависят от выбранного порога классификации. ROC-AUC показывает способность модели ранжировать наблюдения по вероятности положительного класса. Поэтому наиболее убедительными являются случаи, где растут обе группы метрик. Именно поэтому результат BNB на горизонте 1 день является более сильным кандидатом на наличие текстового сигнала, чем случаи, где balanced accuracy растёт при снижении ROC-AUC.

Результат LSTM в основном эксперименте RUN 03 также следует рассматривать через призму проверки признаков. LSTM использовалась как модель, способная учитывать последовательную структуру временного окна \cite{hochreiter1997lstm}. Однако она не улучшила результат относительно более простой логистической регрессии. Это означает, что усложнение архитектуры само по себе не компенсирует ограниченную информативность или нестабильность признакового пространства. Аналогичный вывод встречается и в современных работах по LLM/FinBERT-признакам: более сложная языковая или нейросетевая модель не всегда превосходит простой baseline \cite{shobayo2024sentiment,vuong2025llm}.

Основные ограничения экспериментов можно разделить на несколько групп.

Во-первых, несмотря на расширение в RUN 05, набор активов остаётся ограниченным. Основной эксперимент RUN 03 и дополнительный эксперимент 1 RUN 04 проводились на AAPL, а дополнительный эксперимент 2 RUN 05 --- на нескольких криптоактивах. Поэтому результат нельзя напрямую обобщать на весь фондовый рынок, крипторынок или другие классы активов. В литературе показано, что вклад текстовых данных может различаться между акциями, индексами, криптовалютами и рынками с разной информационной структурой \cite{bollen2011twitter,mao2011sentiment,zou2022prebit}.

Во-вторых, результаты зависят от горизонта прогноза. RUN 05 показывает, что один и тот же актив может давать разный эффект на горизонтах 1, 3 и 7 дней. Это означает, что слабый или сильный результат на одном горизонте не должен интерпретироваться как универсальное свойство текстовых признаков.

В-третьих, в разных экспериментах использовались разные текстовые представления. RUN 03 и RUN 04 опирались на FinBERT embeddings, тогда как RUN 05 использовал более компактные FinBERT-derived sentiment/text-meta признаки. Это методологически допустимо, поскольку работа исследует LLM-generated features как класс признаков, но прямое сравнение величины эффекта между RUN требует осторожности. В дальнейших экспериментах следует отдельно сравнить embeddings, sentiment-score, topic features, emotion features и их комбинации \cite{araci2019finbert,yang2020finbert,albada2025bertweet,vuong2025llm}.

В-четвёртых, качество текстового сигнала зависит от временного выравнивания. В работе используется консервативный подход, снижающий риск look-ahead bias, однако более точная постановка потребовала бы внутридневных timestamps и разделения текстов на опубликованные до и после момента принятия решения. Для криптовалют эта проблема особенно важна, поскольку рынок торгуется непрерывно.

В-пятых, результаты зависят от выбранного протокола оценки. В работе используется хронологическое разбиение без пересечения дат, что методологически строже случайного разбиения. Однако для более устойчивой проверки можно использовать rolling-window validation. В такой схеме модель обучается на окне прошлых данных и тестируется на следующем периоде:

\[
Train_k = [t_0, t_k], \qquad Test_k = (t_k, t_{k+h}],
\]

где $h$ --- длина тестового окна. Такой подход позволил бы проверить, сохраняется ли эффект FinBERT-признаков в разных рыночных режимах.

В-шестых, RUN 05 содержит множественное сравнение активов, горизонтов и групп признаков. Поэтому максимальный найденный прирост может быть завышен относительно среднего эффекта. Для строгой проверки требуется независимый hold-out период, rolling-window validation или поправка на множественное тестирование. В текущей версии работы результат BNB следует рассматривать как сильный кандидат на наличие текстового сигнала, но не как окончательное статистическое доказательство универсальной эффективности текстовых признаков.

В-седьмых, статистическая проверка была проведена только для BNB 1d, то есть для наиболее сильного найденного случая. Чтобы сделать вывод более строгим, аналогичные bootstrap confidence intervals и permutation sanity checks следует применить ко всем активам, горизонтам и группам признаков, а затем учесть множественное сравнение.

Таким образом, результаты работы следует интерпретировать как смешанные. Основной эксперимент RUN 03 показывает слабый текстовый сигнал, дополнительный эксперимент 1 RUN 04 не подтверждает добавочную силу FinBERT embeddings относительно расширенного рыночного baseline для AAPL, а дополнительный эксперимент 2 RUN 05 показывает, что на отдельных криптоактивах, прежде всего BNB, текстовые признаки могут давать существенный прирост. Локальная bootstrap- и permutation-проверка усиливает аргумент в пользу BNB 1d, но не снимает ограничение множественного сравнения. Поэтому основной вывод состоит в том, что LLM/FinBERT-generated features не являются универсально полезными, но могут иметь добавочную предсказательную силу при подходящем активе, источнике текстов и горизонте прогноза.
