\subsection{Дополнительный эксперимент 1: расширение рыночного baseline и снижение размерности}

Для углубления анализа был проведён дополнительный эксперимент RUN 04. Его цель состоит в том, чтобы проверить, зависит ли вывод о добавочной силе FinBERT-признаков от высокой размерности embedding-пространства и от состава рыночного baseline. В отличие от основного эксперимента RUN 03, где сравнение было укрупнённым, RUN 04 разделяет признаки на несколько групп: исходные ценовые признаки, rolling-признаки, признаки торгового объёма, полный FinBERT embedding, PCA-сжатые FinBERT embeddings и объединённые группы признаков.

Эксперимент был построен на данных \texttt{TheFinAI/flare-sm-bigdata} для тикера AAPL. Было извлечено 497 текстов; итоговый датасет содержит 163 наблюдения и 163 уникальные даты за период 2020-03-09 --- 2020-10-27. Хронологическое разбиение имеет вид:

RUN 04 следует рассматривать как exploratory robustness check. Тестовая подвыборка содержит только 25 наблюдений, а в эксперименте сравниваются несколько групп признаков и несколько размерностей PCA. Поэтому точечные значения метрик, включая ROC-AUC, не следует интерпретировать как устойчивые оценки без доверительных интервалов, rolling-window проверки или независимого периода. Основная роль RUN 04 --- проверить, сохраняется ли эффект текстовых признаков при усилении рыночной базовой модели, а не доказать устойчивое превосходство какой-либо конфигурации.

В RUN 04 сохраняется та же постановка целевой переменной:

\[
y_t = \mathbf{1}\{Close_{t+1} > Close_t\}.
\]

Текстовые признаки строятся консервативно: для даты принятия решения $t$ используются только тексты не позднее $t-1$ в окне до 10 дней. Это позволяет снизить риск look-ahead bias.

Проблема полной embedding-размерности сохраняется: FinBERT embedding имеет размерность

\[
e_t \in \mathbb{R}^{768}.
\]

Поэтому в RUN 04 дополнительно строятся PCA-сжатые представления. Пусть $E_{train}$ --- матрица FinBERT embeddings на обучающей подвыборке. После центрирования строится ковариационная матрица

\[
\Sigma = \frac{1}{n_{train}-1}E_{train}^{\top}E_{train}.
\]

Первые $k$ собственных векторов образуют матрицу $V_k$, а сжатое представление имеет вид

\[
e_t^{(k)} = V_k^\top(e_t-\bar e_{train}),
\qquad
k \in \{16,32,64\}.
\]

PCA обучается только на обучающей подвыборке, чтобы избежать утечки информации из валидационной и тестовой подвыборок. После этого одно и то же преобразование применяется к валидационной и тестовой подвыборкам без повторной подгонки. Доля объяснённой дисперсии для $k=16$ составляет 0.984, для $k=32$ --- 0.997, для $k=64$ --- 0.9998. Это означает, что большая часть вариации FinBERT embeddings сжимается в небольшое число компонент, однако сохранение дисперсии не гарантирует сохранение прогнозного сигнала.

На тестовой подвыборке наибольшее значение accuracy и balanced accuracy показывает группа \texttt{market\_full}: accuracy равна 0.560, balanced accuracy равна 0.577, ROC-AUC равен 0.853. Эти точечные значения означают, что расширенные рыночные признаки, включающие rolling-динамику и признаки объёма, в данном запуске дают более сильную рыночную базовую модель, чем исходная группа \texttt{prices\_raw}. Однако из-за малого размера тестовой подвыборки этот результат следует трактовать как exploratory evidence, а не как устойчивую оценку качества.

Добавочная сила PCA-сжатых текстовых признаков относительно \texttt{market\_full} не подтверждается. Для $k=32$ получаем:

\[
\Delta_{text}^{Accuracy}(32) = 0.560 - 0.560 = 0,
\]

\[
\Delta_{text}^{Balanced\ Accuracy}(32) = 0.577 - 0.577 = 0,
\]

\[
\Delta_{text}^{ROC\text{-}AUC}(32) = 0.846 - 0.853 = -0.007.
\]

Для $k=64$ результат хуже:

\[
\Delta_{text}^{Balanced\ Accuracy}(64) = 0.500 - 0.577 = -0.077.
\]

Следовательно, в точечных метриках RUN 04 PCA-сжатые FinBERT embeddings не улучшают результат относительно расширенного рыночного baseline. Текстовые признаки отдельно также не превосходят \texttt{majority baseline}: \texttt{finbert\_full}, \texttt{finbert\_pca\_16}, \texttt{finbert\_pca\_32} и \texttt{finbert\_pca\_64} имеют balanced accuracy 0.500 и ROC-AUC 0.500.

Отдельно следует отметить различие между ROC-AUC и threshold-dependent метриками. У \texttt{market\_full} ROC-AUC равен 0.853, что говорит о способности модели ранжировать наблюдения по вероятности положительного класса в данной тестовой подвыборке. Однако F1 равен только 0.267, а recall составляет 0.154, поскольку подбор на валидационной подвыборке выбрал высокий порог $\tau=0.75$. Поэтому результат RUN 04 лучше интерпретировать как указание на возможный ранжирующий сигнал в расширенных рыночных признаках, но не как устойчиво сильную бинарную классификацию при фиксированном пороге.

Итог RUN 04 формулируется осторожно: в точечных метриках добавочная предсказательная сила FinBERT-generated features относительно \texttt{market\_full} не подтверждается ни в полной embedding-размерности, ни после PCA-сжатия. Это согласуется с выводом RUN 03 о неустойчивости эффекта текстовых признаков, но не является самостоятельным статистическим доказательством из-за малого размера тестовой подвыборки и множественного сравнения групп признаков. Иными словами, эффект RUN 03 не сохраняется при усилении рыночного baseline в данном exploratory-запуске, однако для строгого вывода по RUN 04 требуются доверительные интервалы или rolling-window проверка.
