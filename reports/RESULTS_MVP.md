## 1. Цель эксперимента

Цель MVP-эксперимента — проверить, дают ли признаки, извлечённые из финансовых текстов с помощью FinBERT, дополнительную предсказательную информацию относительно классических рыночных признаков.

Основной вопрос:

```text
Улучшают ли LLM/FinBERT-признаки прогноз направления цены AAPL на следующий торговый день по сравнению с baseline-моделями?
```

---

## 2. Данные и target

В эксперименте использовался один актив — `AAPL`.

Target задан как бинарная переменная направления движения цены закрытия на следующий торговый день:

```text
y_t = 1{Close_{t+1} > Close_t}
```

То есть `y_t = 1`, если цена закрытия следующего торгового дня выше цены закрытия текущего дня, и `y_t = 0` в противном случае.

Эксперимент проводился на daily frequency. Для снижения риска утечки данных текстовые признаки использовались с лагом в один день: для прогноза после дня `t` учитывались тексты не позднее `t-1`.

---

## 3. Split

Основным запуском выбран RUN 03, потому что он одновременно выполняет два условия:

1. Split остаётся хронологическим и не допускает пересечения дат между train, validation и test.
2. Validation и test имеют достаточный размер для первичного сравнения моделей.

Итоговый split:

| Subset | Rows | Unique dates | Period |
|---|---:|---:|---|
| Train | 1104 | 52 | 2020-04-01 — 2020-06-15 |
| Validation | 252 | 10 | 2020-06-16 — 2020-06-29 |
| Test | 222 | 55 | 2020-06-30 — 2020-09-30 |

Split type:

```text
chronological_row_balanced_by_decision_date
```

---

## 4. Feature groups

В эксперименте сравнивались следующие группы признаков:

1. **Prices-only**: табличные рыночные признаки `open`, `high`, `low`, `close`, `adj-close`, `inc-5`, `inc-10`, `inc-15`, `inc-20`, `inc-25`, `inc-30`.
2. **FinBERT-only**: embedding-признаки, извлечённые из финансовых текстов с помощью `ProsusAI/finbert`.
3. **Prices + FinBERT**: объединение рыночных признаков и FinBERT embeddings.

Числовые признаки нормализовались через `StandardScaler`, причём scaler обучался только на train subset. Validation и test использовались только через transform.

---

## 5. Модели

Для MVP были обучены и сопоставлены следующие модели:

| Model | Feature group |
|---|---|
| Majority baseline | none |
| Logistic Regression | prices only |
| Logistic Regression | FinBERT only |
| Logistic Regression | prices + FinBERT |
| LSTM | prices + FinBERT |

Majority baseline всегда предсказывает самый частый класс в train subset и используется как минимальная точка сравнения.

---

## 6. Результаты

| Model | Feature group | Accuracy | Balanced accuracy | F1 | ROC-AUC |
|---|---|---:|---:|---:|---:|
| Majority | none | 0.532 | 0.500 | 0.694 | 0.500 |
| Logistic Regression | prices only | 0.410 | 0.400 | 0.502 | 0.379 |
| Logistic Regression | FinBERT only | 0.550 | 0.529 | 0.667 | 0.512 |
| Logistic Regression | prices + FinBERT | 0.486 | 0.474 | 0.584 | 0.427 |
| LSTM | prices + FinBERT | 0.432 | 0.445 | 0.315 | 0.449 |

Лучший результат среди моделей с FinBERT-признаками показала Logistic Regression на FinBERT-only признаках:

```text
accuracy = 0.550
balanced_accuracy = 0.529
F1 = 0.667
ROC-AUC = 0.512
```

Однако улучшение относительно majority baseline невелико:

```text
Majority accuracy = 0.532
FinBERT-only accuracy = 0.550
```

При этом ROC-AUC для FinBERT-only модели остаётся близким к случайному уровню:

```text
ROC-AUC = 0.512
```

Combined model (`prices + FinBERT`) не улучшает результат относительно FinBERT-only и уступает majority baseline по accuracy. LSTM также не показывает улучшения относительно baseline.

---

## 7. Интерпретация

Результаты MVP не дают оснований утверждать, что FinBERT-признаки обладают устойчивой добавочной предсказательной силой для прогноза next-day direction AAPL в выбранном периоде и при заданном протоколе.

Корректная интерпретация:

```text
В рамках MVP-эксперимента FinBERT-only модель немного превосходит majority baseline по accuracy и balanced accuracy, однако величина прироста мала, ROC-AUC остаётся близким к 0.5, а объединение рыночных и FinBERT-признаков не улучшает результат. Поэтому на текущем этапе нельзя говорить об устойчивом преимуществе LLM-generated features относительно baseline-моделей.
```

---

## 8. Ограничения

1. Эксперимент проведён только на одном активе (`AAPL`).
2. Использован один горизонт прогноза — один торговый день.
3. Период данных ограничен 2020 годом.
4. Использован один основной date-safe split, без rolling-window проверки.
5. FinBERT embeddings использовались как готовые признаки без тонкой настройки модели под конкретную задачу.
6. Результаты чувствительны к периоду, распределению классов и способу агрегации текстов.
