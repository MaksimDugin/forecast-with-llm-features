## 2.5. Модели

В экспериментальной части работы сравниваются несколько моделей, различающихся как по сложности, так и по используемым группам признаков. Такая схема нужна для того, чтобы оценить не только абсолютное качество прогноза, но и добавочную предсказательную силу текстовых признаков, извлечённых с помощью FinBERT. Поэтому каждая модель рассматривается не сама по себе, а относительно baseline-моделей.

В качестве минимальной точки сравнения используется majority baseline. Эта модель не использует ни рыночные, ни текстовые признаки и всегда предсказывает наиболее частый класс в обучающей выборке. В данной задаче такой baseline важен, поскольку распределение классов может быть несбалансированным. Если обучаемая модель показывает качество, близкое к majority baseline, то это означает, что она фактически не извлекает устойчивого сигнала из признаков, а лишь воспроизводит преобладающий класс. Поэтому majority baseline используется как нижняя граница осмысленного качества.

Второй блок моделей основан на логистической регрессии. Логистическая регрессия выбрана как простая и интерпретируемая модель бинарной классификации. В отличие от более сложных нейросетевых архитектур, она позволяет проверить базовую предсказательную силу признаков без риска объяснять результат только сложностью модели. В эксперименте логистическая регрессия обучается отдельно на трёх наборах признаков: только рыночные признаки, только FinBERT-признаки и объединение рыночных и FinBERT-признаков.

Модель prices-only использует только табличные рыночные признаки. Она показывает, насколько хорошо направление цены можно предсказывать на основе исторической ценовой динамики без обращения к текстам. Эта модель служит основным baseline для проверки добавочной силы текстовой информации. Если модель с FinBERT-признаками не превосходит prices-only baseline, то говорить о полезности текстовых признаков в выбранном протоколе нельзя.

Модель FinBERT-only использует только embedding-представления финансовых текстов. Она нужна для отдельной проверки того, содержат ли тексты самостоятельный сигнал для прогноза направления цены. В этом варианте рыночные признаки не используются, поэтому результат модели показывает, насколько информативными оказываются именно текстовые представления.

Модель prices + FinBERT объединяет обе группы признаков. Это ключевой вариант для проверки основной гипотезы работы: если текстовые признаки действительно несут дополнительную информацию, то объединённая модель должна показывать качество выше, чем prices-only и FinBERT-only варианты. Отсутствие такого улучшения означает, что добавление текстовых признаков не даёт устойчивого преимущества в рамках выбранного актива, периода и протокола эксперимента.

Для всех вариантов логистической регрессии используется балансировка классов (`class_weight = balanced`). Это сделано для того, чтобы модель не сводилась к предсказанию только более частого класса. Обучение проводится на train subset, а итоговая оценка качества выполняется на test subset. Validation subset в данном случае используется как часть общего экспериментального протокола и для сопоставимости с нейросетевой моделью.

Дополнительно в работе используется LSTM-модель. Выбор LSTM связан с тем, что входные данные имеют последовательную структуру: для каждого наблюдения формируется окно из нескольких предыдущих дней, а модель получает последовательность признаков. В отличие от логистической регрессии, которая работает с векторизованным представлением наблюдения, LSTM может учитывать порядок элементов внутри временного окна.

Архитектурно LSTM-модель принимает на вход последовательность дневных признаков и возвращает один бинарный прогноз для текущего наблюдения. В эксперименте используется bidirectional LSTM с двумя слоями, dropout-регуляризацией и полносвязным выходным слоем. На выходе модели получается logit, который затем переводится в вероятность класса 1 через сигмоидную функцию. Для обучения используется функция потерь `BCEWithLogitsLoss`, подходящая для бинарной классификации. Для учёта дисбаланса классов в функцию потерь добавляется вес положительного класса.

Обучение LSTM проводится с использованием validation subset. После каждой эпохи считается validation loss; если качество на validation перестаёт улучшаться, применяется early stopping. Такой подход нужен для контроля переобучения: модель не выбирается по минимальной ошибке на train subset, поскольку в этом случае она могла бы запомнить обучающие данные и хуже работать на более позднем периоде. Дополнительно используется планировщик learning rate, уменьшающий скорость обучения при отсутствии улучшения validation loss.

Сравнение majority baseline, логистической регрессии и LSTM позволяет разделить три вопроса. Первый вопрос — превосходит ли модель наивное предсказание наиболее частого класса. Второй — дают ли FinBERT-признаки самостоятельный сигнал по сравнению с рыночными признаками. Третий — улучшает ли более сложная последовательная модель результат относительно простой линейной модели. Такая схема делает эксперимент более устойчивым к завышенной интерпретации: даже если сложная модель показывает ненулевую accuracy, её качество должно сравниваться с простыми baseline.

Качество всех моделей оценивается на test subset с помощью нескольких метрик: accuracy, balanced accuracy, precision, recall, F1-score, ROC-AUC и confusion matrix. Использование нескольких метрик важно, поскольку в задаче бинарного прогноза направления цены accuracy может быть недостаточной, особенно при несбалансированном распределении классов. Balanced accuracy и ROC-AUC позволяют дополнительно оценить, различает ли модель оба класса, а не только воспроизводит преобладающее направление.

Таким образом, набор моделей в работе построен от простого к более сложному: от majority baseline к логистической регрессии и затем к LSTM. Такая организация эксперимента позволяет проверить не только работоспособность отдельных алгоритмов, но и основной исследовательский вопрос: дают ли LLM/FinBERT-признаки измеримую добавочную предсказательную силу относительно рыночных признаков и простых baseline-моделей.
